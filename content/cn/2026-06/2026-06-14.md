### 政策合规

1. **国家网信办发布数据出境安全政策问答。**
国家互联网信息办公室发布 [数据出境安全管理政策法规问答（2026年1月）](https://www.cac.gov.cn/2026-01/30/c_1771505108953002.htm)，明确了个人信息出境标准合同、安全评估、认证三种路径的衔接关系。关键信息基础设施运营者以外的数据处理者，自当年1月1日起累计向境外提供10万人以上、不满100万人个人信息的，可通过标准合同或认证方式出境。超出100万人的，须申报数据出境安全评估。此次问答还特别回应了粤港澳大湾区数据跨境流动标准合同的适用边界，为出海企业提供了清晰的合规指引。

2. **三部委联合启动2026年个人信息保护专项行动。**
中央网信办、工业和信息化部、公安部联合发布 [2026年个人信息保护系列专项行动公告](https://www.kingandwood.com/cn/zh/insights/latest-thinking/changes-and-constants-in-the-special-action-on-personal-information-protection-in-china-2026.html)。此次行动是2025年专项行动的延续和深化，重点聚焦违规收集、超范围使用、非法买卖个人信息等突出问题。监管部门明确表示，2026年将按照"发现一批、整改一批、通报一批、处罚一批"的总体要求推进，形成常态化监管态势。

3. **国家数据局划定2026数据合规7条高压线。**
国家数据局发布新规，明确 [2026年数据合规7项重点要求](https://zhuanlan.zhihu.com/p/2025155352434353945)，涉及数据分类分级、跨境流动、安全审计、事件报告等核心环节。处罚上限提升至5000万元，并同步追究直接负责人责任，形成"企业重罚+个人追责"的双重约束机制。新规覆盖制造、教育、零售等更多行业，不再局限于金融、医疗等传统敏感领域。

4. **"数安中国行"政策宣讲会山东站成功举办。**
国家网信办网络数据管理局指导的 ["数安中国行"2026年数据安全和个人信息保护政策法规宣讲会](https://news.qq.com/rain/a/20260530A0745000) 在济南举行。来自山东省和天津市的党政机关、行业主管部门、重点企业机构约400人参加。会议深入解读了数据出境安全管理、个人信息保护合规审计、数据安全认证等制度标准，标志着法规落地进入实操阶段。

### 漏洞威胁

1. **全球最大学习平台Instructure遭ShinyHunters攻击。**
黑客组织ShinyHunters攻击了在线学习平台Instructure的Canvas系统，窃取约 [3.65TB数据，涉及2.75亿条记录](https://www.pkware.com/blog/2026-data-breaches)，成为教育行业史上最大数据泄露事件。Instructure最终支付了赎金，但安全专家指出"支付赎金并不意味着数据被删除"。该事件暴露了SaaS平台在应对专业勒索团伙时的脆弱性。

2. **2026上半年全球重大数据泄露事件汇总发布。**
安全研究机构HACKMAGEDDON发布 [2026上半年"巨型泄露"报告](https://www.hackmageddon.com/2026/01/29/mega-breaches-in-2026/)，1月至5月间共确认37起重大事件，泄露超过6.52亿条记录。攻击类型中，账户接管（Account Takeover）以12起居首，勒索软件以11起紧随其后。美国是最大受害国（62%的事件发生在美国），教育行业成为黑客首选目标。

3. **CISA新增5个已知被利用漏洞（KEV）。**
据 [2026年6月第1周网络安全形势周报](https://jishuzhan.net/article/2063214811839803393) 显示，CISA本周新增5个已知被利用漏洞，涉及Oracle WebLogic（CVE-2024-21182）、Palo Alto GlobalProtect（CVE-2026-0257）、Android Framework（CVE-2025-48595）等。这些漏洞已被攻击者积极利用，相关企业需尽快修复。

### API 安全

1. **ServiceNow API漏洞致大量客户数据泄露（重点）。**
ServiceNow于6月9日披露了一起 [严重API安全事件](https://www.rescana.com/post/servicenow-api-security-incident-exposes-customer-data-analysis-of-unauthenticated-access-vulnerability-june-2026)。攻击者利用一个未认证的API端点（`/api/now/related_list_edit/create`），在5月1日至6月5日期间对客户实例进行了未授权查询。该漏洞影响了ServiceNow的Australia平台版本及部分早期配置的实例。ServiceNow于6月5日对托管实例实施了安全更新，但联邦机构客户的数据暴露时间长达两个月。安全专家建议所有受影响的客户立即轮换API令牌和服务账户密码。

2. **OWASP API Security Top 10 2026 实施指南更新。**
OWASP发布了 [2026年版API Security Top 10实施指南](https://imperialis.tech/en/blog/api-security-hardening-2026-owasp-top-10-implementation-guide)，BOLA（Broken Object Level Authorization）仍以40%的攻击占比稳居首位。新指南特别强调了GraphQL API的防护、影子API（Shadow API）治理以及AI驱动的自动攻击检测。值得关注的是，组织平均有超过20%的API端点属于"影子API"——未记录、未受监管的端点，成为新的攻击面。指南建议企业建立完整的API资产清单，实施细粒度的授权检查，并部署实时的API行为监控。

3. **Vercel OAuth供应链攻击事件复盘。**
安全研究机构Guardz深度分析了 [Vercel 4月安全事件](https://guardz.com/blog/vercel-data-breach-driven-by-oauth-compromise)，攻击者通过入侵第三方AI工具Context.ai的Google Workspace OAuth应用，劫持了Vercel员工的账号并进入内部系统。这起事件再次证明OAuth集成是供应链攻击的关键突破口。研究人员建议企业限制OAuth应用的权限范围、实施定期权限审计、为敏感环境变量启用加密存储。

### 行业动向

1. **2026年数据安全监管进入"持证合规"时代。**
据 [行业分析报告](https://www.sohu.com/a/960974577_122493512)，2026年网络安全合规迎来系统性升级。CCRC（中国网络安全审查认证）认证成为企业上市前数据安全合规的权威凭证，数据安全官（DSO）、数据安全评估（DSA）、数据合规官（DCO）等认证体系加速完善。金融、医疗、教育等行业纷纷将合规认证纳入供应商准入条件。

2. **诺和诺德临床试验数据泄露事件曝光。**
丹麦制药巨头诺和诺德于6月11日披露了一起 [网络攻击事件](https://www.studioglobal.ai/zh-cn/discover/answers/what-happened-in-the-june-2026-cyberattack-6a2b5e1a8f7e203b138258ca)，影响部分临床试验参与者的假名化数据，包括生物标志物、出生年份和生活方式因素。公司表示不包含直接身份标识信息，仅影响有限的内部IT系统，未波及生产、供应链或商业活动。此次事件再次引发对医疗数据安全的关注。

3. **国务院立法计划揭示网络安全合规三大趋势。**
据 [立法计划解读](https://hea.china.com/articles/20260520/202605201873767.html)，《网络安全等级保护条例》进入立法关键窗口期，《互联网信息服务管理办法》修订将适应生成式AI带来的监管挑战，《网络犯罪防治法草案》进一步明确实名制要求。三大趋势表明，2026年数据安全监管正向"全流程、全链路、全场景"方向演进。

### 开源工具

1. **API安全扫描工具推荐：Nuclei + 自定义模板。**
开源漏洞扫描器Nuclei支持用户编写 [自定义API安全测试模板](https://github.com/projectdiscovery/nuclei)，可用于检测OWASP Top 10中的常见API漏洞，包括BOLA、Broken Authentication、SSRF等。社区已贡献超过100个API专用检测模板，覆盖REST、GraphQL、gRPC等协议。结合CI/CD流水线，可实现API安全的自动化左移。

2. **Mimoto：开源API安全网关。**
近期受到关注的 [Mimoto项目](https://github.com/mimoto/mimoto) 是一个轻量级的开源API安全网关，支持OAuth 2.0 / OIDC集成、速率限制、请求验证和审计日志。该项目通过插件化架构支持自定义安全策略，并提供了与Kubernetes原生集成的Ingress Controller，适合云原生环境下的API安全防护。

### 社媒分享

1. **安全研究员演示ServiceNow API漏洞利用过程。**
多位安全研究员在X平台上分享了 [ServiceNow API漏洞的技术细节](https://www.triskelelabs.com/resources/servicenow-security-incident-unauthenticated-api-access-exposing-customer-data)，指出该漏洞的核心是REST端点缺少认证检查。BleepingComputer报道称，从4月22日漏洞通过赏金计划提交到6月5日补丁部署，中间存在长达6周的暴露窗口。社区呼吁厂商缩短漏洞修复周期。

2. **"攻击者不再破解，而是直接登录"——5月安全事件启示。**
安全分析师指出， [5月重大安全事件](https://www.pkware.com/blog/2026-data-breaches) 揭示了一个残酷现实：攻击者不再需要复杂的漏洞利用，社会工程已成为主要入口。Charter Communications（Spectrum）因一名员工被vishing攻击导致490万客户数据泄露；Carnival Corporation因单个社交工程攻击导致近600万客户护照信息暴露。安全社区呼吁企业将防御重心从"防漏洞"转向"防社工"。
