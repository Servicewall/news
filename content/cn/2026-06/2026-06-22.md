## 🏛️ 政策合规

1. 📜 **微软 6 月补丁星期二创纪录：修复近 200 个安全漏洞**  
   微软发布 2026 年 6 月安全更新，共修补近 200 个漏洞，刷新该公司单月补丁数量纪录。涉及 Windows、Office、Edge 等产品，其中多个漏洞已被野外利用。建议企业 IT 部门立即评估并部署更新。  
   [阅读原文](https://krebsonsecurity.com/2026/06/a-record-breaking-patch-tuesday-for-june-2026/)

2. 🧾 **SAML 认证新脆弱性分析：绕过技术曝光**  
   PortSwigger 研究员公开了针对 Ruby 和 PHP 版 SAML 生态的完整认证绕过方法，利用属性污染、命名空间混淆等解析器不一致问题，可导致单点登录（SSO）被完全击穿。身份提供商需紧急自查配置。  
   [阅读原文](https://portswigger.net/research/the-fragile-lock)

3. 🚩 **国内暂无新规发布，但多部门持续关注 AI 数据安全**  
   截至本日，国内未出台新的数据安全或 API 安全专项政策。但工信部、网信办近期多次强调 AI 大模型与 API 接口的数据跨境合规问题，预计下半年将有细化指引出台。  
   *（注：国内政策暂无具体新闻，此为趋势性提醒）*

## 🕳️ 漏洞威胁

1. 🎬 **FFmpeg 修复“PixelSmash”漏洞，可致远程代码执行**  
   FFmpeg 发布安全更新，修复代号 PixelSmash 的严重漏洞（CVE 暂未分配）。攻击者可通过特制视频文件在 Jellyfin 等视频服务器上触发远程代码执行或拒绝服务。所有使用 FFmpeg 的流媒体服务应尽快升级。  
   [阅读原文](https://www.bleepingcomputer.com/news/security/ffmpeg-fixes-pixelsmash-flaw-in-widely-used-video-decoder/)

2. 🤖 **Popa 安卓僵尸网络与以色列上市公司关联，影响百万电视盒子**  
   连续四年活跃的 Popa 僵尸网络强制数百万台消费级电视盒子代理广告欺诈、账户接管和数据窃取流量。KrebsOnSecurity 调查发现其与一家公开交易的以色列公司存在联系。国内部分电视盒子用户也需警惕。  
   [阅读原文](https://krebsonsecurity.com/2026/06/popa-botnet-linked-to-publicly-traded-israeli-firm/)

3. 💰 **JaredFromSubway MEV 机器人遭 1500 万美元加密货币盗窃**  
   以太坊上知名 MEV 机器人 JaredFromSubway 被攻击者利用虚假交易对操控其机会检测逻辑，导致 1500 万美元损失。此类针对交易机器人的攻击手法正在演化，API 安全防护需考虑交易逻辑层面风险。  
   [阅读原文](https://www.bleepingcomputer.com/news/security/jaredfromsubway-mev-bot-hacked-in-15-million-crypto-theft/)

## 🔐 API 安全

1. 🤖 **Dify 平台曝“DifyTap”漏洞：AI 聊天记录可在租户间泄露**  
   研究人员披露开源 AI 工作流平台 Dify（GitHub 14.6 万星）的四个高危漏洞，攻击者可静默访问其他租户的 AI 对话历史、窃取 API 密钥甚至执行任意代码。使用 Dify 的企业需立即升级至最新版本。  
   [阅读原文](https://thehackernews.com/2026/06/researchers-detail-difytap-flaws-in.html)

2. 🧪 **PortSwigger 发布 2025 年度 Top 10 网页攻击技术**  
   第 19 届社区评选结果出炉，涵盖 HTTP 请求走私、OAuth 滥用、GraphQL 注入等 Web/API 攻击手法。其中多类技术可直接用于 API 端点绕过与数据泄露，安全团队应将其纳入渗透测试重点。  
   [阅读原文](https://portswigger.net/research/top-10-web-hacking-techniques-of-2025)

3. 🔓 **SAML 认证绕过技术（续）：API 网关配置风险**  
   在 Fragile Lock 研究中，SAML 解析器不一致问题同样影响基于 API 的 SSO 集成。攻击者通过构造特殊断言可绕过 RESTful API 的认证检查，建议所有使用 SAML 的 API 网关立即校验签名与命名空间。  
   [阅读原文](https://portswigger.net/research/the-fragile-lock)
