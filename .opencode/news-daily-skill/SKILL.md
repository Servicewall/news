# 安全洞察日报 - 手动生成工作流

## 适用场景

当 GitHub Actions 自动发布失败、当日新闻需要人工审核、或需要临时创建日报时使用。

---

## 工作流步骤

### Step 1: 联网搜索新闻

搜索今日数据安全 / API 安全相关新闻，国内源占 **60%**，国际源占 **40%**。

**国内搜索关键词（8-10 条候选）：**
```
数据安全 数据泄露 API安全 政策法规 合规 2026年6月
数据出境 个人信息保护 数据分类分级 等保
数据安全 安全漏洞 2026年
```

**国际搜索关键词（5-6 条候选）：**
```
data breach API security vulnerability 2026 june
OWASP top 10 API CVE exploited ransomware
cybersecurity data leak sql injection 2026
```

**推荐新闻源：**
- 国内：安全内参 secrss.com、国家网信办 cac.gov.cn、FreeBuf freebuf.com、嘶吼 4hou.com
- 国际：BleepingComputer、The Hacker News、CISA、OWASP Blog、PortSwigger Research

### Step 2: 按板块归类

从搜索结果中选 **12-15 条**，按以下 6 个板块归类，控制配比：

| 板块 | 国内（约 7-9 条） | 国外（约 5-6 条） |
|---|---|---|
| **① 政策合规** | 2-3 条（政策解读、法规动态、监管行动） | 0-1 条（国际监管动态） |
| **② 漏洞威胁** | 1-2 条（国内数据泄露、漏洞通报） | 1-2 条（国际重大泄露、CVE） |
| **③ API 安全** | 0-1 条 | 1-2 条（API 漏洞、网关安全） |
| **④ 行业动向** | 1-2 条（投融资、报告、趋势） | 1 条 |
| **⑤ 开源工具** | 0-1 条 | 1 条（推荐 GitHub 安全项目） |
| **⑥ 社媒分享** | — | 1-2 条（X/Twitter 安全讨论） |

### Step 3: 编写 Markdown 文件

#### 文件路径
```

```


### Step 4: 本地预览

```bash
cd /Users/donge/workspace/news
hugo server
```

打开浏览器访问 `http://localhost:1313/news/`，检查：
- 首页卡片是否正确显示新日报
- 日报内容排版正常（链接可点、emoji 显示、代码块格式正确）
- 配色符合深蓝+红色警示主题

### Step 5: 提交推送

```bash
git add content/cn/
git commit -m "daily: security report YYYY-MM-DD"
git push
```

推送后 `.github/workflows/pages.yaml` 会自动构建并部署到：
**https://servicewall.github.io/news/**

## 写作规范

| 规则 | 说明 |
|---|---|
| **每条长度** | 80-150 字 |
| **原文链接** | 每条必带，格式 `[关键词](url)` |
| **emoji 风格** | 安全主题：🔴 🚨 📡 🔒 ⚠️ 🛡️ 🔑 🔓 |
| **国内内容** | 以政策合规、数据安全事件、监管动态为主 |
| **国际内容** | 以 API 安全、漏洞利用、开源工具、社工攻击为主 |
| **标题级别** | `###` 作为 6 个板块分隔 |
| **禁止表格** | 移动端排版不友好，用列表代替 |
| **摘要代码块** | 用 ``` 包裹，5-6 行，不加语言标识 |
| **配图** | 非必须；如需配图用 Markdown 标准 `![alt](url)` |

## 常见问题

**Q: 新闻不够 12 条怎么办？**
A: 最少 8 条即可成刊，优先保证政策合规 + API 安全两个核心板块。

**Q: 临时发现严重安全事件如何插入？**
A: 放在对应板块的**第一条**，并更新今日摘要中的对应行。

**Q: 国外新闻找不到怎么办？**
A: 适当降低国外占比至 30%，或从国内安全厂商的海外事件分析报道中提取。
